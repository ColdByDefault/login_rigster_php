Convention
PHP
- Dateinamen: Verwenden Sie das lower-Camel-Case für Dateinamen (z.B. my_script.php | myScripte.php).
- Klassen und Funktionen: Verwenden Sie PascalCase für Klassennamen und camelCase für Methodennamen (z.B. MyClass, myFunction()).
- Variablen: Verwenden Sie camelCase für Variablen (z.B. $myVariable).
- Einrückung: Verwenden Sie vier Leerzeichen für die Einrückung.
- Klammern: Öffnende Klammern für Funktionen und Kontrollstrukturen sollten in derselben Zeile stehen, schließende Klammern in der neuen Zeile.
- Dokumentation: Verwenden Sie PHPDoc-Kommentare, um Klassen, Methoden und Funktionen zu dokumentieren.
- Struktur: Halten Sie Klassen und Funktionen klein und fokussiert.
- Namespaces: Verwenden Sie Namespaces für die Organisation von Klassen und Funktionen.
- Fehlerbehandlung: Verwenden Sie Ausnahmen (try-catch) für die Fehlerbehandlung.
HTML
- DOCTYPE: Verwenden Sie immer eine doctype-Deklaration, z.B. <!DOCTYPE html>.
- Kleinbuchstaben: Verwenden Sie Kleinbuchstaben für HTML-Tags und -Attribute.
- Attribute: Fügen Sie keine unnötigen Attribute hinzu und verwenden Sie Anführungszeichen für alle Attributwerte.
- Einrückung: Verwenden Sie zwei Leerzeichen für die Einrückung.
- Semantisches Markup: Verwenden Sie semantische Tags (<header>, <footer>, <article>, <section>, etc.), um die Struktur des Inhalts zu kennzeichnen.
- Schließende Tags: Verwenden Sie immer schließende Tags für alle Elemente, die sie benötigen (außer bei selbstschließenden Tags wie <br />).
- Whitespace: Halten Sie eine konsistente Verwendung von Leerzeilen zwischen logischen Abschnitten des Codes ein.
JavaScript
- Variablen: Verwenden Sie let und const anstelle von var für die Deklaration von Variablen.
- Funktionen: Verwenden Sie camelCase für Funktionsnamen (z.B. myFunction()).
- Einrückung: Verwenden Sie zwei Leerzeichen für die Einrückung.
- Klammern: Öffnende Klammern für Funktionen und Kontrollstrukturen sollten in derselben Zeile stehen, schließende Klammern in der neuen Zeile.
- Semikolons: Verwenden Sie Semikolons am Ende jeder Anweisung.
- Ereignisbindung: Verwenden Sie addEventListener() anstelle von Inline-Event-Handlern.
- Dokumentation: Kommentieren Sie den Code mit JSDoc oder einfachen Kommentaren, um die Lesbarkeit zu verbessern.
- Funktionale Programmierung: Nutzen Sie funktionale Programmieransätze wie map, filter und reduce, wo möglich.
